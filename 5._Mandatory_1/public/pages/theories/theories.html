Variable identifiers - var vs const vs let

var - The scope of a var is either a Global Variable when declared outside of a function. but var can also be declared inside a functions scope and both updated and re-declared. This can become an issue when you have multiple var's with the same name in the same function scope

let - The scope of a let is block scoped which is bound by curly brackets {}. A let can be updated however it cannot be redefined.

const - The scope of a const is block scoped which is bound by curly brackets {}. A const cannot be updated nor redefined.

Hoisting - The useful part of hoisting is when functions are declared after they are called. the variables const and let however wont be initialized with a default value when hoisted to the top, where var will be initiallized with undefined.

Design of a REST API

REST API Memes
HTTP    Endpoints       Description
GET     "/memes"        Gets all memes
GET     "/memes/{id}"   Gets meme by specific ID
POST    "/memes"        Adds a new meme
PUT     "/memes/{id}"   Replaces entire meme object by an id with an entire new meme object
PATCH   "/memes/{id}"   Replaces part of a meme object by an id.
DELETE  "/memes/{id}"   Deletes a meme by an ID
The REST API uses Full CRUD (Create, Read, Update, Delete).

Node.js

Node.js is a server side JavaScript runtime environment designed to build network applications by executing JavaScript code outside a web browser. Node.js uses npm as a package manager which is used to run scripts, publish projects, install packages and much more

Server Side Rendering

Server Side Rendering also known as SSR is useful to render full HTML for a page for imporved performance. This also helps to avoid CORS-errors with data fetching with client side rendering. Useful for: Improved performance, Faster load times & better social sharing
